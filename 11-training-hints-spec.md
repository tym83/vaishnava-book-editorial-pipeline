# Training Hints Spec v1

## Назначение

`training hints` нужны для управляемого усиления `docx_semantic_style_classifier.py` на реальном корпусе.

Они не заменяют эвристики, а:

- подавляют ложные срабатывания;
- задают точные стили для известных абзацев;
- переводят спорные места в `review_only`;
- позволяют постепенно обучать pipeline на вручную размеченных главах.

---

## Формат

Файл: `json`

Структура:

```json
{
  "input": "/path/to/docx",
  "rules": [
    {
      "match": { "type": "index", "value": 185 },
      "action": "review_only",
      "style": "Источник",
      "note": "abbrev-reference-line"
    }
  ]
}
```

---

## Поля

### `match.type`

Допустимые значения:

- `index`
- `text`
- `regex`

### `match.value`

- для `index`: номер абзаца в `word/document.xml`
- для `text`: точный текст абзаца
- для `regex`: регулярное выражение по исходному тексту абзаца

### `action`

Допустимые значения:

- `set_style`
- `review_only`
- `ignore`

### `style`

Используется для:

- `set_style`
- `review_only`

Обычно это один из канонических стилей:

- `Шлока`
- `Перевод шлоки`
- `Цитата 1`
- `Цитата 2`
- `Письмо`
- `Источник`
- `Подпись к иллюстрации`

### `note`

Свободный комментарий для редактора.

---

## Семантика действий

### `set_style`

Классификатор сразу применяет указанный стиль.

### `review_only`

Классификатор не применяет стиль автоматически, но выносит абзац в `review` уже с указанным стилем и пометкой из `note`.

### `ignore`

Классификатор не трогает этот абзац и не выносит его в `review`.

---

## Приоритет

Hints применяются **раньше** эвристик.

Порядок:

1. `index`
2. `text`
3. `regex`

Если правило совпало, обычные эвристики для этого абзаца уже не запускаются.

---

## Workflow

### 1. Подготовить docx

```text
unicode_normalizer
-> style_normalizer
```

### 2. Выгрузить шаблон hints

```bash
python3 docx_semantic_style_classifier.py export-hints-template input.docx hints.json
```

### 3. Разметить `hints.json` вручную

Менять:

- `action`
- `style`
- `note`

### 4. Запустить classifier с hints

```bash
python3 docx_semantic_style_classifier.py classify input.docx output.docx --hints-json hints.json
```

### 5. Проверить результат

- `report-json`
- `report-md`
- `docx_style_audit`

---

## Практический совет

На первых итерациях лучше:

- не пытаться размечать всё;
- размечать только повторяющиеся ложные срабатывания;
- отдельно размечать устойчивые типы:
  - список книг;
  - справочник сокращений;
  - строки-источники;
  - подписи;
  - письма;
  - короткие standalone quotes.

Это даст быстрый рост качества без ручной правки тысяч абзацев.
