# GitHub Repository Setup

This file contains a practical GitHub-facing description for the project.

## Suggested Repository Name

`vaishnava-book-editorial-pipeline`

## Suggested Short Description

Chapter-based automation toolkit for translating, reviewing, proofreading, and typesetting Vaishnava books with `DOCX`, `Vedabase`, and `InDesign`.

## Suggested Topics

- `python`
- `docx`
- `indesign`
- `editorial-workflow`
- `translation`
- `proofreading`
- `typesetting`
- `vedabase`
- `digital-humanities`

## Suggested README Pitch

Use this repository when you need:

- structured chapter splitting;
- deterministic editorial QA;
- scripture-reference cleanup;
- review bundles that can be applied in `Word`, `PDF`, or `InDesign`;
- a bridge from editorial text preparation to layout QA.

## Suggested Visibility

Choose based on corpus sensitivity:

- `Private`, if working files or publisher materials are included.
- `Public`, only if repository contents are limited to tooling, docs, and safe sample artifacts.

## Before Publishing Publicly

Check that the repository does not contain:

- copyrighted source books;
- publisher-internal working files;
- private reviewer notes that should not be public;
- local mirror data that should stay private.
