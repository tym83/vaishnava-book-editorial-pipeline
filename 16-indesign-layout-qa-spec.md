# InDesign Layout QA Spec v1

## 1. Скрипт

- [indesign_layout_qa.jsx](/home/tym83/Загрузки/Служение/Automate/scripts/indesign_layout_qa.jsx)

Среда:

- `InDesign 2022`
- `JSX / ExtendScript`

---

## 2. Что делает v1

Скрипт:

1. открывает `INDD` или берет активный документ;
2. при включенном safe-mode делает только механически безопасные правки;
3. пересобирает композицию;
4. собирает layout QA report;
5. при необходимости сохраняет документ.

При этом v1 теперь пишет не только markdown-report, но и machine-readable JSON, который можно дальше конвертировать в issue bundle для PDF annotations или InDesign notes.

---

## 3. Что считается safe fixes

В `v1` автоматически правятся только такие вещи:

- неразрывный пробел после безопасных однобуквенных русских предлогов и союзов:
  - `в`
  - `к`
  - `с`
  - `у`
  - `и`
  - `о`
  - `а`
  - `я`
- неразрывный пробел между числом и `%`;
- неразрывный пробел между `№` и номером;
- неразрывный пробел перед тире в обычной фразе, чтобы строка не начиналась с тире после перелома;
- неразрывный пробел в типовых ссылках вида:
  - `рис. 3`
  - `табл. 4`
  - `гл. 2`
  - `стр. 15`
  - `прим. 7`
- включение переносов в рабочих текстовых стилях;
- выключение переносов в заголовках и некоторых служебных стилях.

Что `v1` не делает молча:

- не меняет знаки препинания по смыслу;
- не пытается автоматически лечить widows/orphans;
- не переписывает front matter;
- не чинит вручную style overrides;
- не меняет язык текста автоматически.

## 4. Какие стили он ожидает

По умолчанию скрипт считает body-style-слоями:

- `Основной текст`
- `Цитата 1`
- `Цитата 2`
- `Письмо`
- `Источник`
- `Сноска 1`
- `Сноска 2`
- `Сноска 3`
- `Сноска 4`
- `Список нумерованный 1`
- `Список нумерованный 2`
- `Список ненумерованный 1`
- `Список ненумерованный 2`
- `Перевод шлоки`

И style-слоями без переносов:

- `Заголовок 1`
- `Заголовок 2`
- `Заголовок 3`
- `Заголовок 4`
- `Источник`
- `Подпись к иллюстрации`

Оба списка можно передавать через `app.scriptArgs`.

---

## 5. Что пишет report

В report попадает:

- путь к документу;
- был ли включен safe-fix режим;
- был ли документ сохранен;
- сколько сделано safe replacements;
- на каких стилях включены переносы;
- на каких стилях переносы выключены;
- `overset frames`;
- `missing fonts`;
- `hyphenation style issues`;
- `paragraph overrides`;
- `character overrides`;
- `language issues`;
- `legacy symbol issues`;
- `footnote issues`;
- `dangling dash suspects`;
- `widow/orphan suspects`;
- `front matter flags`.

По умолчанию report пишется как:

- `*.layout-qa-report.md`
- `*.layout-qa-report.json`

рядом с `.indd`.

## 6. Что значат основные секции

### Overset Frames

Нужно руками проверить все перечисленные frame/story.

### Missing Fonts

Это блокирующая проблема перед финальной сдачей PDF.

### Hyphenation Style Issues

Показывает стили, где ожидаемое состояние переносов не совпало с фактическим.

### Paragraph / Character Overrides

Это индикатор ручного форматирования поверх стилей. Такие места нужно spot-check.

### Language Issues

Проверяются только диапазоны, где есть кириллица. Скрипт не переписывает язык, он только сигналит.

### Legacy Symbol Issues

Тут ловятся:

- private-use символы;
- остатки `Gaura`-подобных шрифтов.

### Footnote Issues

В `v1` это прежде всего:

- сноски без переносов;
- потенциально неготовые к нормальной композиции footnote paragraphs.

### Dangling Dash Suspects

Это список строк, где не первая строка абзаца начинается с:

- `—`
- `–`

Обычно это признак того, что тире переползло на новую строку и требует проверки.

### Widow/Orphan Suspects

Это эвристика, а не окончательный вердикт. Проверять глазами в макете обязательно.

### Front Matter Flags

Это только structural diagnostics:

- не найден `Заголовок 1`;
- первый `Заголовок 1` начинается подозрительно поздно;
- слишком много `Основной текст` до первого `Заголовок 1`.

## 7. Как запускать

### Вариант 1. На активном документе

Открыть нужный `INDD` и запустить скрипт.

### Вариант 2. Через `app.scriptArgs`

Поддерживаются параметры:

- `input`
- `report`
- `report_json`
- `apply_safe_fixes`
- `save_after`
- `russian_language_names`
- `body_styles`
- `no_hyphen_styles`

---

## 8. Рекомендуемый порядок

1. Подготовить `docx`.
2. Импортировать его через `word_to_indesign_import.jsx`.
3. Открыть получившийся `indd`.
4. Прогнать `indesign_layout_qa.jsx` с `apply_safe_fixes=true`.
5. Проверить report.
6. Руками пройти:
   - overset;
   - widows/orphans;
   - language issues;
   - overrides;
   - suspicious footnotes.

---

## 9. Ограничения v1

1. Скрипт не аннотирует макет цветом и не ставит InDesign notes.
2. `Widow/Orphan Suspects` строятся по строкам и страницам, поэтому возможны false positives.
3. `Language Issues` опираются на applied language у text ranges и требуют ручной валидации.
4. Скрипт не лечит сложные композиционные решения front matter.
5. Полноценный тест возможен только на машине с установленным `InDesign`.
