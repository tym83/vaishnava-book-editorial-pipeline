# BBT Style Rules Backing `manual_bbt_v1`

Это короткая сводка правил, которые вручную перенесены из материалов в `~/Загрузки/корректура BBT`.

## Приоритет источников

1. BBT rule docs
2. утвержденные русские книги корпуса
3. Бхакти Викаша Свами как референсный корпус
4. локальный `vedabase` RU

## Курсив

Основа:

- санскритские термины обычно курсивом;
- имена собственные обычно прямым;
- `вайшнав` прямым;
- `авайшнав` курсивом;
- сочетания типа `Кали-юга`, `Гаудия-сампрадая`, `Шеша-аватара` прямым;
- `кришна-прасад` может оставаться курсивом;
- если слово курсивом, по умолчанию оно со строчной кроме начала предложения.

Источник:

- `Курсив.doc`
- `Слова и словосочетания.doc`

## Диакритика

Основа:

- в прозе по умолчанию без диакритики;
- в транслитерации и полноценных санскритских цитатах диакритика сохраняется;
- одиночные термины в русской прозе обычно без диакритики;
- если слово уже склоняется по-русски, диакритика снимается.

Источник:

- `Транслитерация и знаки диакритики.doc`

## Прописные и строчные

Основа:

- религиозные местоимения и имена Господа по контексту с прописной;
- слово в курсиве обычно со строчной;
- `Шри` в именах с пробелом а не через дефис;
- части составных имен и титулов ориентируются на BBT name rules.

Локальные house overrides:

- используем формы `Гурудев` и `Гуру Махарадж`;
- `Гуру Махарадж` пишем раздельно;
- слитные и hyphen variants вроде `Гурумахарадж` и `Гуру-Махарадж` не считаем каноническими словарными формами.

Источник:

- `Прописные буквы.doc`
- `Имена собственные.doc`

## Географические названия

Основа:

- `дхама`, `тиртха`, `ашрам`, `гхат` в географических названиях обычно со строчной;
- `дхама` через дефис и без курсива;
- `Пури` и подобные компоненты часто через дефис и с прописной в составных именах;
- `Вриндаван`, `Майяпур`, `Матхура`, `Навадвипа`, `Джаганнатха-Пури` берутся по BBT form.

Источник:

- `Индийские географические названия.doc`

## Что автоматизировано

- `stylistic_reviewer.py` проверяет `discouraged_forms`;
- `docx_style_audit.py` проверяет только safe cases по `italic_automation`;
- `conditional` случаи остаются в ручном review.
